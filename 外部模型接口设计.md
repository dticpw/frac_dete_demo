# 外部模型接口设计

本文档记录 `frac_dete_demo` 为后续接入其它部位骨折模型、骨结构分割模型或外部参考模型预留的接口设计。

## 1. 设计目标

当前 demo 暂时没有可直接用于腕、足、踝、肘等部位的可靠骨折检测模型。

后续可能接入的模型包括：

- 肋骨骨折 CT 模型；
- 颈椎/脊柱骨折模型；
- 骨结构分割模型；
- 医院数据训练出的本项目模型；
- 其它候选生成器。

这些模型的输出形式可能不同，有的是分类，有的是分割，有的是检测框，有的是点位候选。为了避免后续在 `app.py` 中写大量临时逻辑，当前先预留统一 adapter 接口。

相关代码位于：


## 3. 统一输出格式

所有外部模型或候选生成器最终都应转换为 `ExternalCandidate`：

```python
ExternalCandidate(
    source="heuristic",
    candidate_id="1_edge_0492",
    case_id="1",
    slice_index=492,
    x=147,
    y=52,
    z=492,
    score=0.82,
    label="weak_candidate",
    note="sharp bone-window edge change",
    status="unreviewed",
)
```

字段含义：

| 字段 | 含义 |
| --- | --- |
| `source` | 候选来源，例如 `heuristic`、`ribfrac_reference` |
| `candidate_id` | 候选点唯一编号 |
| `case_id` | 病例编号 |
| `slice_index` | 轴位切片号 |
| `x` | 图像左右方向坐标 |
| `y` | 图像上下方向坐标 |
| `z` | 体数据 z 坐标 |
| `score` | 当前来源给出的排序分或参考分 |
| `label` | 候选标签或模型输出类别 |
| `note` | 对候选来源、限制或原因的说明 |
| `status` | 人工审核状态 |

注意：不同模型的 `score` 不一定可直接比较。跨模型融合前需要校准。

## 4. Adapter 协议

每个 adapter 应实现：

```python
class SomeAdapter:
    name = "some_source"
    display_name = "Some Source"
    enabled = True
    out_of_domain = False

    def predict(self, case_id, volume_hu, metadata=None):
        ...
        return list[ExternalCandidate]
```

其中：

- `name`：程序内部使用的来源名；
- `display_name`：页面或日志中展示的名称；
- `enabled`：是否默认启用；
- `out_of_domain`：是否属于跨部位参考模型；
- `predict()`：接收 CT 体数据，返回统一候选点列表。

## 5. 当前已实现 Adapter

### 5.1 HeuristicAdapter

文件：

```text
src/external_models/heuristic_adapter.py
```

状态：

```python
enabled = True
out_of_domain = False
```

作用：

- 包装当前启发式候选检测算法；
- 内部调用 `detect_candidates()`；
- 将 `Candidate` 转换为 `ExternalCandidate`；
- 当前页面实际展示的候选点仍来自它。

### 5.2 外部参考模型占位 Adapter

文件：

```text
src/external_models/placeholders.py
```

当前占位：

- `RibFractureReferenceAdapter`
- `SpineFractureReferenceAdapter`
- `BoneSegmentationReferenceAdapter`

状态：

```python
enabled = False
out_of_domain = True
```

它们暂时不返回任何候选点，只作为后续接入位置。

## 6. Registry

文件：

```text
src/external_models/registry.py
```

主要函数：

```python
get_all_adapters()
get_enabled_adapters()
run_enabled_adapters(case_id, volume_hu, metadata=None)
```

当前 `app.py` 不再直接调用 `detect_candidates()`，而是调用：

```python
run_enabled_adapters(...)
```

这样后续新增模型时，只需要：

1. 新建 adapter 文件；
2. 在 `registry.py` 中注册；
3. 设置 `enabled = True` 或在页面中提供开关。

## 7. 后续接入外部模型的建议

如果接入肋骨或脊柱模型，不建议把输出当作可靠诊断结果，而应标注为：

```text
out_of_domain_reference
```

推荐输出说明：

```python
note="Rib fracture model reference; not trained for wrist/foot CT."
```

页面上应单独提示：

> 该结果来自跨部位参考模型，不代表当前部位可靠诊断。

## 8. 后续可能的页面改造

当前页面仍使用原来的 `Weak Candidates` 表格。

后续可以扩展为：

- 增加 `source` 列；
- 增加 `label` 列；
- 增加模型来源筛选；
- 增加启用/关闭 adapter 的复选框；
- 将跨域参考模型输出放到 `External Model References` 区域；
- 区分 `heuristic`、`manual`、`external_reference`、`trained_model`。

## 9. 当前版本的边界

当前只是预留接口和保存架构版本：

- 没有下载或集成任何外部模型权重；
- 没有运行肋骨/脊柱模型；
- 页面输出仍主要来自启发式规则；
- 外部模型占位 adapter 默认关闭；
- 还没有做跨模型分数校准。

这一步的价值是：后续可以按同一 adapter 规范逐步接入外部参考模型，而不破坏现有 demo 主流程。
