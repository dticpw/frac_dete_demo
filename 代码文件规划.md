# Fracture Detection Demo 代码文件规划

## 1. 环境判断

当前默认环境 `th123` 可以支撑第一版 demo。

已具备的关键依赖：

- Python 3.10.19
- `pydicom 3.0.2`
- `numpy 2.2.6`
- `scipy 1.15.2`
- `pandas 2.3.3`
- `scikit-image 0.25.2`
- `opencv-python-headless 4.10.0.84`
- `matplotlib 3.10.8`
- `gradio 6.9.0`
- `torch 2.7.1+cu118`

暂缺但第一版可不使用的依赖：

- `SimpleITK`
- `MONAI`
- `nibabel`
- `plotly`
- `streamlit`

结论：第一版 demo 不需要安装新包。先用 `pydicom + numpy/scipy + skimage/opencv + matplotlib + gradio` 实现 DICOM 读取、三视图预览、骨窗增强、启发式候选提示和标注导出。后续如果进入训练或更强医学影像处理阶段，再评估是否安装 `SimpleITK`、`MONAI`、`nibabel`。

## 2. 第一版 Demo 目标

第一版优先做出可演示闭环：

- 从 `../测试/` 读取 5 个 DICOM 病例；
- 自动排序切片并堆叠为 3D CT volume；
- 显示轴位、冠状位、矢状位三视图；
- 支持骨窗显示；
- 输出病例摘要；
- 用启发式规则生成疑似候选区域；
- 点击候选后跳转到对应切片；
- 支持医生确认/否定/手动添加候选；
- 导出标注 JSON/CSV。

## 4. 文件职责

### `app.py`

Gradio demo 入口。

负责：

- 加载病例列表；
- 管理当前病例和当前切片位置；
- 显示三视图；
- 显示候选区域列表；
- 接收医生确认、否定和新增标注；
- 调用导出逻辑。

### `src/config.py`

集中管理路径和默认参数。

建议包含：

- 项目根目录；
- DICOM 测试数据目录：`../测试`;
- 样例图片目录：`../样例图片`;
- 解析预览目录：`../解析预览`;
- 标注输出目录；
- 默认窗宽窗位；
- 候选检测阈值。

### `src/dicom_loader.py`

DICOM 读取与体数据构建。

负责：

- 递归查找 DICOM 文件；
- 读取元数据；
- 按 `InstanceNumber` 或 `ImagePositionPatient` 排序；
- 提取像素数组；
- 应用 `RescaleSlope` 和 `RescaleIntercept` 得到 HU；
- 返回 `volume` 和 `metadata`。

### `src/volume_processing.py`

体数据预处理。

负责：

- 裁剪 HU 范围；
- 骨窗归一化；
- 简单去噪；
- 提取骨阈值 mask；
- 必要时做最大连通域或小连通域分析。

### `src/view_rendering.py`

三视图渲染。

负责：

- 根据当前 index 渲染 axial/coronal/sagittal；
- 叠加候选框或候选点；
- 输出 Gradio 可显示的 RGB 图像；
- 生成局部候选缩略图。

### `src/candidate_detection.py`

第一版启发式候选检测。

候选来源：

- 高 HU 小连通域；
- 骨结构边缘局部突变；
- 疑似皮质中断区域；
- 局部高梯度异常。

输出格式建议：

```python
{
    "candidate_id": "case1_cand_0001",
    "case_id": "1",
    "view": "axial",
    "slice_index": 130,
    "x": 312,
    "y": 245,
    "z": 130,
    "score": 0.72,
    "reason": "small high-density fragment near bone edge"
}
```

注意：这里的 `score` 只是启发式排序分数，不是医学模型置信度。

### `src/annotation_store.py`

标注保存与读取。

负责：

- 保存医生确认结果；
- 保存误报；
- 保存手动新增点位；
- 导出 JSON；
- 导出 CSV；
- 后续可转换为训练集标注格式。

### `src/case_index.py`

病例索引和摘要。

负责：

- 扫描 `../测试/` 下病例；
- 读取 `../解析预览/dicom_summary.csv`；
- 为 app 提供病例列表；
- 输出病例基本信息，例如切片数、尺寸、像素间距、层厚。

## 5. 实现顺序

建议按下面顺序推进：

1. 创建基础目录和空模块；
2. 实现 `dicom_loader.py`，确保 5 个病例都能读成 volume；
3. 实现 `view_rendering.py`，先显示单病例三视图；
4. 实现 `case_index.py`，支持病例切换；
5. 实现 `app.py` 的基础 Gradio 页面；
6. 加入骨窗和切片滑块；
7. 实现简单候选检测；
8. 加入候选列表和点击跳转；
9. 实现医生确认/否定/新增标注；
10. 导出标注 JSON/CSV。

## 6. 后续扩展位置

当医院愿意提供更多数据后，可以新增：

```text
src/model/
  dataset.py
  train_patch_classifier.py
  infer_patch_classifier.py

```

后续训练路线：

- 小数据：patch classifier；
- 中等数据：3D candidate classifier；
- 数据和标注充足：MONAI / nnU-Net 分割或检测模型。
