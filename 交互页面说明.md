# 交互页面说明

本文档说明 `frac_dete_demo` 第一版交互页面的使用方式，以及页面中各个区域、图片和候选编号的含义。

## 1. 页面整体定位

当前页面是骨折检测 demo 的第一版原型，主要用途是：

- 浏览 DICOM 多层 CT 病例；
- 用三视图查看 CT 体数据；
- 用骨窗方式突出骨结构；
- 展示程序自动生成的疑似候选点；
- 让医生或使用者确认、否定或手动添加标记；
- 导出后续可用于训练的数据。

需要注意：当前候选点来自启发式规则，不是已经训练好的临床模型。候选分数只用于排序，不代表医学置信度。

## 2. 病例选择区

页面顶部有：

- `Case` 下拉框；
- `Load Case` 按钮。

下拉框中类似：

```text
Case 1 (520 slices)
Case 2 (633 slices)
Case 3 (520 slices)
...
```

含义是：

- `Case 1`：对应项目目录中的 `测试\1`；
- `520 slices`：该病例有 520 张 CT DICOM 切片。

选择病例后，页面会读取该病例的 DICOM 序列，并堆叠成一个三维 CT 体数据。

首次加载某个病例可能需要几秒，因为程序需要读取几百张 DICOM 切片。

## 3. Case Summary

`Case Summary` 显示当前病例的基本信息，例如：

```text
Case: 1
Path: E:\PG\fracture_detection\测试\1
Slices: 520
Size: 644 x 644
Pixel spacing: 0.25\0.25
Slice thickness: 0.25 mm
Volume shape: 520 x 644 x 644
Candidate score is heuristic only.
```

字段含义：

- `Path`：该病例 DICOM 文件所在目录；
- `Slices`：切片数量；
- `Size`：单张 CT 切片的像素尺寸；
- `Pixel spacing`：平面内每个像素对应的实际物理尺寸；
- `Slice thickness`：相邻切片方向的层厚；
- `Volume shape`：程序内部三维数组形状，顺序为 `z x y x x`。

## 4. 三个 CT 图像窗口代表什么

页面中间有三个图像：

- `Axial`
- `Coronal`
- `Sagittal`

它们是同一个三维 CT 体数据的三个观察方向。

### 4.1 Axial

`Axial` 是轴位图，也就是原始 CT 切片方向。

可以理解为从头到脚或沿扫描方向一层一层看。

在当前数据里，`Axial z` 滑块控制显示第几张 CT 切片。

例如：

```text
Axial z = 260
```

表示当前显示第 260 层轴位切片。

### 4.2 Coronal

`Coronal` 是冠状位重建图。

可以理解为从身体前后方向切开后看到的视图。

它不是原始 DICOM 里的单张图片，而是程序从三维体数据中重建出来的截面。

`Coronal y` 滑块控制冠状位截面在 y 方向的位置。

### 4.3 Sagittal

`Sagittal` 是矢状位重建图。

可以理解为从身体左右方向切开后看到的视图。

它同样是程序从三维体数据中重建出来的截面。

`Sagittal x` 滑块控制矢状位截面在 x 方向的位置。

## 5. 三个滑块是什么意思

页面有三个位置滑块：

```text
Axial z
Coronal y
Sagittal x
```

它们共同表示当前在三维 CT 体数据中的观察位置：

- `z`：第几张轴位切片；
- `y`：图像上下方向的位置；
- `x`：图像左右方向的位置。

当你移动某一个滑块时，对应方向的图像会变化。

如果选中某个候选点，页面会自动把这三个滑块跳转到候选点所在的位置。

## 6. Window Center 和 Window Width

页面有两个窗宽窗位滑块：

- `Window Center`
- `Window Width`

它们控制 CT 图像的显示方式。

当前默认值是骨窗：

```text
Window Center = 500
Window Width = 2000
```

这个设置会让骨结构更清楚，适合观察骨皮质、骨片和疑似骨折线。

简单理解：

- `Window Center` 控制显示亮度中心；
- `Window Width` 控制灰度范围宽窄；
- 宽度越小，对比越强，但容易丢失细节；
- 宽度越大，灰度范围更宽，但对比可能降低。

## 7. 红色标记是什么意思

如果某张图上出现红色十字或红色框，表示当前切面附近有一个程序提示的疑似候选点。

红色标记不是最终诊断结论，只表示程序认为该位置值得人工检查。

目前红色标记可能来自：

- 小的高密度骨片；
- 骨结构边缘的强烈变化；
- 局部可能存在皮质中断；
- 其他基于规则的异常提示。

## 8. Weak Candidates 表格

`Weak Candidates` 是疑似候选点列表。

每一行代表一个程序生成的候选点。

表格字段包括：

| 字段 | 含义 |
| --- | --- |
| `candidate_id` | 候选点编号 |
| `slice_index` | 候选点所在轴位切片号 |
| `x` | 候选点在图像左右方向的位置 |
| `y` | 候选点在图像上下方向的位置 |
| `score` | 启发式排序分数 |
| `reason` | 程序给出该候选的原因 |
| `status` | 人工审核状态 |

## 9. candidate_id 怎么看

候选编号类似：

```text
1_edge_0492
1_frag_0007
```

它由几部分组成。

### 9.1 `1_edge_0492`

含义：

```text
1      -> 病例编号，也就是 Case 1
edge   -> 候选来源类型
0492   -> 相关切片号或内部序号
```

因此：

```text
1_edge_0492
```

表示：

> 在 Case 1 中，程序基于骨结构边缘突变规则，在大约第 492 层附近发现了一个候选点。

### 9.2 `1_frag_0007`

含义：

```text
1      -> 病例编号，也就是 Case 1
frag   -> small fragment，疑似小骨片规则
0007   -> 该连通区域的内部编号
```

因此：

```text
1_frag_0007
```

表示：

> 在 Case 1 中，程序基于小高密度连通区域规则，发现了第 7 个疑似小骨片候选。

## 10. reason 字段是什么意思

当前常见的 `reason` 有：

```text
sharp bone-window edge change
small high-density component
manual marker
```

含义如下：

| reason | 含义 |
| --- | --- |
| `sharp bone-window edge change` | 骨窗下局部边缘变化很强，可能是皮质边缘、断裂、伪影或正常骨边界 |
| `small high-density component` | 检测到小的高密度连通区域，可能是小骨片，也可能是正常结构或噪声 |
| `manual marker` | 使用者手动添加的标记 |

这些原因是为了帮助人工理解程序为什么提示该点，不等于诊断依据。

## 11. score 字段是什么意思

`score` 是启发式排序分数。

它的作用是让程序把更“显眼”的候选点排在前面。

当前不能把它理解成：

```text
骨折概率
AI 置信度
临床可信度
```

更准确的理解是：

```text
规则认为这个点值得优先看的程度
```

例如：

```text
score = 0.82
```

只表示该点在当前启发式规则下比较突出。

## 12. 如何查看某个候选点

操作步骤：

1. 在 `Weak Candidates` 表格中点击某一行；
2. 页面会自动更新 `Axial z`、`Coronal y`、`Sagittal x`；
3. 三个图像会跳转到该候选点附近；
4. 查看红色标记周围的骨结构；
5. 判断是否确实像骨折或异常。

如果图像没有明显变化，可以稍微拖动附近的 `Axial z` 滑块，查看候选点上下几层。

骨折通常需要结合连续多层观察，不能只看单张切片。

## 13. 三个审核按钮

页面下方有：

```text
Confirm First Candidate
Reject First Candidate
Add Manual Marker At Current Crosshair
Export Annotations
```

### 13.1 Confirm First Candidate

确认选中的候选点。

当前版本按钮文字还保留了 `First Candidate`，但实际逻辑已经改为：

> 先点击表格中的某一行，再点击确认按钮，程序会把该行状态改成 `confirmed`。

后续可以把按钮文字改成 `Confirm Selected Candidate`。

### 13.2 Reject First Candidate

否定选中的候选点。

同样需要先点击表格中的某一行。

点击后，该行状态会改成：

```text
rejected
```

### 13.3 Add Manual Marker At Current Crosshair

在当前三个滑块对应的位置添加一个人工标记。

适用场景：

- 程序没有提示某个可疑位置；
- 人工在浏览 CT 时发现了疑似骨折；
- 需要把这个位置记录下来。

添加后，表格里会出现类似：

```text
1_manual_0008
```

表示 Case 1 中新增的人工标记。

### 13.4 Export Annotations

导出当前候选点和人工审核状态。

导出位置在：

```text
frac_dete_demo\data\annotations\
```

会同时保存：

- JSON 文件；
- CSV 文件。

这些文件后续可以作为训练数据整理的基础。

## 14. 推荐使用流程

建议按下面流程使用：

1. 选择病例；
2. 等待病例加载完成；
3. 先查看 `Case Summary`，确认切片数和体数据尺寸；
4. 浏览 `Axial` 图像，大致了解扫描范围；
5. 查看 `Weak Candidates` 表格；
6. 点击候选点，跳转到对应位置；
7. 在三个视图中综合判断；
8. 如果认可候选，点击确认；
9. 如果明显误报，点击否定；
10. 如果发现程序漏掉的位置，拖动滑块到目标点后添加人工标记；
11. 全部检查后导出标注。

## 15. 当前版本局限

当前版本只是第一版可运行 demo，有明显局限：

- 还没有真正训练过骨折检测模型；
- 候选点来自简单规则，误报会比较多；
- 没有自动识别具体解剖部位；
- 没有做复杂的三维骨结构分割；
- 没有医学级测量工具；
- 没有窗宽窗位预设按钮；
- 标注工具还比较简陋；
- 候选点不代表最终诊断结论。

这些局限正是后续需要通过医院标注数据和模型训练逐步解决的内容。

## 16. 后续页面改进建议

后续可以优先改进：

- 把按钮文字从 `First Candidate` 改成 `Selected Candidate`；
- 在图像上显示当前坐标和切片号；
- 增加候选缩略图；
- 增加上一候选/下一候选按钮；
- 增加窗宽窗位预设，例如骨窗、软组织窗；
- 增加标注备注字段；
- 增加医生姓名或审核人字段；
- 增加病例级结论字段；
- 增加导出训练集格式的功能。
