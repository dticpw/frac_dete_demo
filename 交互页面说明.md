# 交互页面说明

本文档说明 `frac_dete_demo` 第一版交互页面的使用方式，以及页面中各个区域、图片和候选编号的含义。

## 1. 页面整体定位

当前页面是骨折检测 demo 的第一版原型，主要用途是：

- 浏览 DICOM 多层 CT 病例；
- 用三视图查看 CT 体数据；
- 用骨窗方式突出骨结构；
- 展示程序自动生成的疑似候选点；
- 让医生或使用者确认、否定或手动添加标记；
- 导出后续可用于训练的数据。

需要注意：当前候选点来自启发式规则，不是已经训练好的临床模型。候选分数只用于排序，不代表医学置信度。

## 2. 病例选择区

页面顶部有：

- `Case` 下拉框；
- `Load Case` 按钮。

下拉框中类似：

```text
Case 1 (520 slices)
Case 2 (633 slices)
Case 3 (520 slices)
...
```

含义是：

- `Case 1`：对应项目目录中的 `测试\1`；
- `520 slices`：该病例有 520 张 CT DICOM 切片。

选择病例后，页面会读取该病例的 DICOM 序列，并堆叠成一个三维 CT 体数据。

首次加载某个病例可能需要几秒，因为程序需要读取几百张 DICOM 切片。

## 3. Case Summary

`Case Summary` 显示当前病例的基本信息，例如：

```text
Case: 1
Path: E:\PG\fracture_detection\测试\1
Slices: 520
Size: 644 x 644
Pixel spacing: 0.25\0.25
Slice thickness: 0.25 mm
Volume shape: 520 x 644 x 644
Candidate score is heuristic only.
```

字段含义：

- `Path`：该病例 DICOM 文件所在目录；
- `Slices`：切片数量；
- `Size`：单张 CT 切片的像素尺寸；
- `Pixel spacing`：平面内每个像素对应的实际物理尺寸；
- `Slice thickness`：相邻切片方向的层厚；
- `Volume shape`：程序内部三维数组形状，顺序为 `z x y x x`。

## 4. 三个 CT 图像窗口代表什么

页面中间有三个图像：

- `Axial`
- `Coronal`
- `Sagittal`

它们是同一个三维 CT 体数据的三个观察方向。

### 4.1 Axial

`Axial` 是轴位图，也就是原始 CT 切片方向。

可以理解为从头到脚或沿扫描方向一层一层看。

在当前数据里，`Axial z` 滑块控制显示第几张 CT 切片。

例如：

```text
Axial z = 260
```

表示当前显示第 260 层轴位切片。

### 4.2 Coronal

`Coronal` 是冠状位重建图。

可以理解为从身体前后方向切开后看到的视图。

它不是原始 DICOM 里的单张图片，而是程序从三维体数据中重建出来的截面。

`Coronal y` 滑块控制冠状位截面在 y 方向的位置。

### 4.3 Sagittal

`Sagittal` 是矢状位重建图。

可以理解为从身体左右方向切开后看到的视图。

它同样是程序从三维体数据中重建出来的截面。

`Sagittal x` 滑块控制矢状位截面在 x 方向的位置。

### 4.4 为什么能从一组轴位切片重建出冠状位和矢状位

这里使用的是医学影像中常见的多平面重建技术，英文通常叫：

```text
MPR: Multi-Planar Reconstruction
```

CT 扫描得到的并不只是互不相关的一张张图片。每张 DICOM 切片都有自己的空间位置、像素间距和层厚信息。把这些切片按正确顺序堆叠起来后，就得到一个三维体数据，可以理解为一个由很多小体素组成的 3D 灰度数组。

当前程序内部的体数据形状类似：

```text
z x y x x
```

其中：

- `z`：轴位切片方向；
- `y`：单张图像的上下方向；
- `x`：单张图像的左右方向。

`Axial` 图像本质上是：

```python
volume[z, :, :]
```

`Coronal` 图像本质上是：

```python
volume[:, y, :]
```

`Sagittal` 图像本质上是：

```python
volume[:, :, x]
```

所以冠状位和矢状位不是额外拍摄出来的，而是从同一个 CT 体数据中按不同方向重新切出来的截面。

这也是为什么它看起来像“另外拍了一次 CT”：因为 CT 数据本身已经包含了三维空间信息，只是原始 DICOM 通常按轴位切片保存。

### 4.5 这样重建出来的图像准确吗

在切片排序、像素间距、层厚和方向信息都正确的情况下，MPR 重建是医学影像中非常常规、可靠的显示方式。医院影像工作站里看到的冠状位、矢状位重建，本质上也是类似思路。

不过当前 demo 里的实现是简化版，有一些需要注意的地方：

- 当前主要按切片顺序堆叠，没有完整处理所有 DICOM 空间方向矩阵；
- 当前没有做复杂的各向同性重采样；
- 如果 z 方向层厚和 x/y 平面像素间距不同，重建图像的比例可能和真实物理比例不完全一致；
- 如果 DICOM 序列存在倾斜扫描、方向矩阵复杂、切片间距不均匀，简单堆叠会不够严谨；
- 当前 Coronal 和 Sagittal 是直接从体数据取截面，主要用于 demo 浏览，不等同于完整医学工作站级 MPR；
- 对细微骨折线的判断仍然要结合原始轴位切片和多层连续观察。

当前这批数据的像素间距和层厚大多接近，例如 `0.25 mm x 0.25 mm x 0.25 mm` 或 `0.30 mm x 0.30 mm x 0.30 mm`，因此重建效果会比较自然，看起来也更接近真实的等比例三维视图。

如果后续要做更严谨的医学级 MPR，可以引入 `SimpleITK` 这类医学影像库，根据 DICOM 的空间方向、origin、spacing 做标准重采样和方向校正。

## 5. 三个滑块是什么意思

页面有三个位置滑块：

```text
Axial z
Coronal y
Sagittal x
```

它们共同表示当前在三维 CT 体数据中的观察位置：

- `z`：第几张轴位切片；
- `y`：图像上下方向的位置；
- `x`：图像左右方向的位置。

当你移动某一个滑块时，对应方向的图像会变化。

如果选中某个候选点，页面会自动把这三个滑块跳转到候选点所在的位置。

## 6. Window Center 和 Window Width

页面有两个窗宽窗位滑块：

- `Window Center`
- `Window Width`

它们控制 CT 图像的显示方式。

当前默认值是骨窗：

```text
Window Center = 500
Window Width = 2000
```

这个设置会让骨结构更清楚，适合观察骨皮质、骨片和疑似骨折线。

简单理解：

- `Window Center` 控制显示亮度中心；
- `Window Width` 控制灰度范围宽窄；
- 宽度越小，对比越强，但容易丢失细节；
- 宽度越大，灰度范围更宽，但对比可能降低。

## 7. 红色标记是什么意思

如果某张图上出现红色十字或红色框，表示当前切面附近有一个程序提示的疑似候选点。

红色标记不是最终诊断结论，只表示程序认为该位置值得人工检查。

目前红色标记可能来自：

- 小的高密度骨片；
- 骨结构边缘的强烈变化；
- 局部可能存在皮质中断；
- 其他基于规则的异常提示。

## 8. Weak Candidates 表格

`Weak Candidates` 是疑似候选点列表。

每一行代表一个程序生成的候选点。

表格字段包括：

| 字段 | 含义 |
| --- | --- |
| `candidate_id` | 候选点编号 |
| `slice_index` | 候选点所在轴位切片号 |
| `x` | 候选点在图像左右方向的位置 |
| `y` | 候选点在图像上下方向的位置 |
| `score` | 启发式排序分数 |
| `reason` | 程序给出该候选的原因 |
| `status` | 人工审核状态 |

## 9. candidate_id 怎么看

候选编号类似：

```text
1_edge_0492
1_frag_0007
```

它由几部分组成。

### 9.1 `1_edge_0492`

含义：

```text
1      -> 病例编号，也就是 Case 1
edge   -> 候选来源类型
0492   -> 相关切片号或内部序号
```

因此：

```text
1_edge_0492
```

表示：

> 在 Case 1 中，程序基于骨结构边缘突变规则，在大约第 492 层附近发现了一个候选点。

### 9.2 `1_frag_0007`

含义：

```text
1      -> 病例编号，也就是 Case 1
frag   -> small fragment，疑似小骨片规则
0007   -> 该连通区域的内部编号
```

因此：

```text
1_frag_0007
```

表示：

> 在 Case 1 中，程序基于小高密度连通区域规则，发现了第 7 个疑似小骨片候选。

## 10. reason 字段是什么意思

当前常见的 `reason` 有：

```text
sharp bone-window edge change
small high-density component
manual marker
```

含义如下：

| reason | 含义 |
| --- | --- |
| `sharp bone-window edge change` | 骨窗下局部边缘变化很强，可能是皮质边缘、断裂、伪影或正常骨边界 |
| `small high-density component` | 检测到小的高密度连通区域，可能是小骨片，也可能是正常结构或噪声 |
| `manual marker` | 使用者手动添加的标记 |

这些原因是为了帮助人工理解程序为什么提示该点，不等于诊断依据。

## 11. score 字段是什么意思

`score` 是启发式排序分数。

它的作用是让程序把更“显眼”的候选点排在前面。

当前不能把它理解成：

```text
骨折概率
AI 置信度
临床可信度
```

更准确的理解是：

```text
规则认为这个点值得优先看的程度
```

例如：

```text
score = 0.82
```

只表示该点在当前启发式规则下比较突出。

## 12. 如何查看某个候选点

操作步骤：

1. 在 `Weak Candidates` 表格中点击某一行；
2. 页面会自动更新 `Axial z`、`Coronal y`、`Sagittal x`；
3. 三个图像会跳转到该候选点附近；
4. 查看红色标记周围的骨结构；
5. 判断是否确实像骨折或异常。

如果图像没有明显变化，可以稍微拖动附近的 `Axial z` 滑块，查看候选点上下几层。

骨折通常需要结合连续多层观察，不能只看单张切片。

## 13. 3D Bone Preview 是什么

`3D Bone Preview` 是一个降采样后的骨结构三维预览。

它的生成方式是：

1. 后端读取完整 CT 体数据；
2. 按骨阈值提取高密度骨结构；
3. 对体数据做降采样，降低浏览器负担；
4. 用表面重建算法生成轻量 OBJ 网格；
5. 把候选点作为小几何标记一起写入模型。

它的作用是帮助理解候选点在整体骨结构中的空间位置。

需要注意：

- 3D 预览不是完整精度 CT；
- 3D 预览不适合判断细微骨折线；
- 细节判断仍然要看 `Axial`、`Coronal`、`Sagittal` 三个 CT 视图；
- 第一次加载某个病例时会生成 3D 缓存，之后会直接读取缓存。
- 当前 3D 预览会将最大体数据边长降采样到约 260 以内，以控制浏览器加载开销。
- Case 2 的原始图像尺寸和覆盖范围更大，如果过度降采样，会更容易出现类似“粘土块”的外观；当前版本已经提高了 3D 预览精度。

## 14. nnInteractive Segmentation Reference

`nnInteractive Segmentation Reference` 是交互式 3D 分割参考区域。

它的作用不是自动判断骨折，而是在你指定一个点之后，让 nnInteractive 根据这个点生成一个局部 3D mask。当前页面先展示 axial 方向的 mask 叠加图，后续可以继续接入 3D mask 高亮。

使用方式：

1. 先加载病例；
2. 在 `Axial` 图像上点击一个骨结构或疑似区域；
3. 页面会记录该点的 `z, y, x` 坐标；
4. 点击 `Run Interactive Segmentation`；
5. 等待模型运行完成；
6. 查看 `nnInteractive Axial Mask Overlay` 中的橙色区域；
7. 查看 `nnInteractive Status` 中的耗时、mask voxel 数和输出路径。

当前输出位置：

```text
frac_dete_demo\outputs\nninteractive_gradio\
```

注意：

- 这是点提示分割，不是骨折检测；
- mask 的范围取决于点击位置；
- 如果点击到骨外软组织，结果可能没有参考意义；
- 当前官方权重 license 为 `CC BY-NC-SA 4.0`，正式部署或商业交付前需要重新核对授权；
- 页面中显示的 mask 可以作为医生标注或 3D 高亮的基础，但不能直接作为诊断结论。

## 15. 日常展示时如何挂起 Gradio 服务

如果只是自己临时查看，可以直接前台运行：

```powershell
D:/python/anaconda/envs/fracmed/python.exe app.py
```

这种方式会占用当前 PowerShell 窗口。关闭窗口或按 `Ctrl+C` 后，服务会停止。

如果需要把服务挂在后台用于展示，可以在项目目录执行：

```powershell
cd E:\PG\fracture_detection\frac_dete_demo

$log = Join-Path (Get-Location) "gradio_7860.log"
$err = Join-Path (Get-Location) "gradio_7860.err.log"

Start-Process `
  -FilePath "D:/python/anaconda/envs/fracmed/python.exe" `
  -ArgumentList @("app.py") `
  -WorkingDirectory (Get-Location) `
  -RedirectStandardOutput $log `
  -RedirectStandardError $err `
  -WindowStyle Hidden
```

启动后打开：

```text
http://127.0.0.1:7860
```

检查服务是否已经挂起成功：

```powershell
Get-NetTCPConnection -LocalPort 7860 -ErrorAction SilentlyContinue |
  Select-Object LocalAddress,LocalPort,State,OwningProcess
```

如果看到 `Listen`，说明服务正在运行。`OwningProcess` 是进程 PID。

查看后台日志：

```powershell
Get-Content gradio_7860.log -Tail 80
Get-Content gradio_7860.err.log -Tail 80
```

停止后台服务：

```powershell
$pid7860 = (Get-NetTCPConnection -LocalPort 7860 -ErrorAction SilentlyContinue |
  Where-Object { $_.State -eq "Listen" } |
  Select-Object -First 1 -ExpandProperty OwningProcess)

if ($pid7860) {
  Stop-Process -Id $pid7860
}
```

注意：

- 展示前建议先确认 `http://127.0.0.1:7860` 能打开；
- 如果 7860 已被占用，需要先停止旧服务，或临时修改 `app.py` 里的端口；
- nnInteractive 首次运行可能需要下载权重，正式展示前应提前跑一次；
- 展示时不要把 nnInteractive 说成自动骨折检测，它是点提示分割参考。

## 16. 审核按钮

页面下方有：

```text
Confirm Selected Candidate
Reject Selected Candidate
Add Manual Marker At Current Crosshair
Export Annotations
```

### 16.1 Confirm Selected Candidate

确认选中的候选点。

需要先点击表格中的某一行，再点击确认按钮，程序会把该行状态改成 `confirmed`。

### 16.2 Reject Selected Candidate

否定选中的候选点。

同样需要先点击表格中的某一行。

点击后，该行状态会改成：

```text
rejected
```

### 16.3 Add Manual Marker At Current Crosshair

在当前三个滑块对应的位置添加一个人工标记。

适用场景：

- 程序没有提示某个可疑位置；
- 人工在浏览 CT 时发现了疑似骨折；
- 需要把这个位置记录下来。

添加后，表格里会出现类似：

```text
1_manual_0008
```

表示 Case 1 中新增的人工标记。

### 16.4 Export Annotations

导出当前候选点和人工审核状态。

导出位置在：

```text
frac_dete_demo\data\annotations\
```

会同时保存：

- JSON 文件；
- CSV 文件。

这些文件后续可以作为训练数据整理的基础。

## 17. 推荐使用流程

建议按下面流程使用：

1. 选择病例；
2. 等待病例加载完成；
3. 先查看 `Case Summary`，确认切片数和体数据尺寸；
4. 浏览 `Axial` 图像，大致了解扫描范围；
5. 查看 `Weak Candidates` 表格；
6. 点击候选点，跳转到对应位置；
7. 在三个视图中综合判断；
8. 如果认可候选，点击确认；
9. 如果明显误报，点击否定；
10. 如果发现程序漏掉的位置，拖动滑块到目标点后添加人工标记；
11. 如需查看局部分割参考，在 `Axial` 图像点击疑似区域并运行 nnInteractive；
12. 全部检查后导出标注。

## 18. 当前版本局限

当前版本只是第一版可运行 demo，有明显局限：

- 还没有真正训练过骨折检测模型；
- 候选点来自简单规则，误报会比较多；
- 没有自动识别具体解剖部位；
- 3D 预览是降采样骨结构表面，不是完整精度 CT 体渲染；
- nnInteractive 是交互式分割参考，不是自动骨折检测模型；
- 没有医学级测量工具；
- 没有窗宽窗位预设按钮；
- 标注工具还比较简陋；
- 候选点不代表最终诊断结论。

这些局限正是后续需要通过医院标注数据和模型训练逐步解决的内容。
