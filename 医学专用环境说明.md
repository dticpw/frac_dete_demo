# 医学专用环境说明

本项目优先使用医学影像专用 conda 环境：

```text
fracmed
```

解释器路径：

```text
D:/python/anaconda/envs/fracmed/python.exe
```

该环境从 `th123` 克隆而来，然后额外安装了医学影像相关依赖，避免污染通用环境。

## 1. 创建方式

由于 conda 从 defaults 下载时遇到 SSL 问题，本环境使用本地已有 `th123` 克隆创建：

```powershell
D:/python/anaconda/Scripts/conda.exe create -n fracmed --clone th123 -y
```

## 2. 已安装的医学影像包

额外安装命令：

```powershell
D:/python/anaconda/envs/fracmed/python.exe -m pip install totalsegmentator SimpleITK nibabel plotly
```

关键包包括：

- `TotalSegmentator 2.14.0`
- `SimpleITK 2.5.5`
- `nibabel 5.4.2`
- `plotly 6.8.0`
- `nnunetv2 2.8.0`
- `dicom2nifti 2.6.2`
- `python-gdcm 3.2.6`
- `torch 2.7.1+cu118`
- `gradio 6.9.0`
- `pydicom 3.0.2`

`pip check` 当前结果：

```text
No broken requirements found.
```

## 3. Demo 启动方式

推荐：

```powershell
.\run_demo_fracmed.ps1
```

或直接：

```powershell
D:/python/anaconda/envs/fracmed/python.exe app.py
```

## 4. 使用原则

- 当前项目后续医学影像模型实验优先使用 `fracmed`；
- 不再向 `th123` 安装 TotalSegmentator、MONAI、nnU-Net 等医学专用大包；
- 如果只是普通 Python/文档任务，可以不启动环境；
- 如果后续接入 TotalSegmentator 或其它医学模型，默认使用 `fracmed`。

## 5. 注意事项

安装时 pip 提示若干命令行脚本不在 PATH 中，例如：

```text
TotalSegmentator.exe
nnUNetv2_predict.exe
```

这不影响项目使用。需要调用时应使用绝对路径，或者通过：

```powershell
D:/python/anaconda/envs/fracmed/python.exe -m <module>
```

后续如果需要直接在终端中使用这些命令，可以先激活环境：

```powershell
D:/python/anaconda/Scripts/conda.exe activate fracmed
```

### 6.1 Windows / scikit-image IO 补丁

当前机器上 `from skimage import io` 会触发 Windows 原生异常，并导致 `nnunetv2` 的 image reader registry 导入失败。TotalSegmentator 的 NIfTI 推理并不需要这个 natural image reader，但 nnU-Net 顶层会导入它。

因此 `fracmed` 环境中对以下文件做了本地补丁：

```text
D:/python/anaconda/envs/fracmed/Lib/site-packages/nnunetv2/imageio/natural_image_reader_writer.py
```

补丁内容：

- 移除顶层 `from skimage import io`；
- 改为在读写 PNG/TIFF 时懒加载 `imageio.v3`。

如后续重装 `nnunetv2` 导致补丁丢失，可重新执行：

```powershell
D:/python/anaconda/envs/fracmed/python.exe scripts/patch_nnunet_skimage_io.py
```

验证：

```powershell
D:/python/anaconda/envs/fracmed/python.exe -c "import nnunetv2.inference.predict_from_raw_data; print('ok')"
```

### 6.2 License 限制

`appendicular_bones` 任务最贴近手腕、足部、踝、肘等四肢骨结构，但 TotalSegmentator 当前提示该任务不是开放任务，需要 license。

因此在没有 license 前：

- 不能直接使用 `appendicular_bones` 作为页面中的骨结构专家；
- 可以测试开放的 `total` 任务，但它只覆盖部分大骨，如 humerus、femur、clavicula、scapula 等；
- 对腕骨、跗骨、掌骨、跖骨等小骨，开放 `total` 任务并不够用。

### 6.3 当前探针结果
