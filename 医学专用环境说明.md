# 医学专用环境说明

本项目优先使用医学影像专用 conda 环境：

```text
fracmed
```

解释器路径：

```text
D:/python/anaconda/envs/fracmed/python.exe
```

该环境从 `th123` 克隆而来，然后额外安装了医学影像相关依赖，避免污染通用环境。

## 1. 创建方式

由于 conda 从 defaults 下载时遇到 SSL 问题，本环境使用本地已有 `th123` 克隆创建：

```powershell
D:/python/anaconda/Scripts/conda.exe create -n fracmed --clone th123 -y
```

## 2. 已安装的医学影像包

额外安装命令：

```powershell
D:/python/anaconda/envs/fracmed/python.exe -m pip install totalsegmentator SimpleITK nibabel plotly
```

关键包包括：

- `TotalSegmentator 2.14.0`
- `SimpleITK 2.5.5`
- `nibabel 5.4.2`
- `plotly 6.8.0`
- `nnunetv2 2.8.0`
- `dicom2nifti 2.6.2`
- `python-gdcm 3.2.6`
- `torch 2.7.1+cu118`
- `gradio 6.9.0`
- `pydicom 3.0.2`

`pip check` 当前结果：

```text
No broken requirements found.
```

## 3. Demo 启动方式

推荐：

```powershell
.\run_demo_fracmed.ps1
```

或直接：

```powershell
D:/python/anaconda/envs/fracmed/python.exe app.py
```

## 4. 使用原则

- 当前项目后续医学影像模型实验优先使用 `fracmed`；
- 不再向 `th123` 安装 TotalSegmentator、MONAI、nnU-Net 等医学专用大包；
- 如果只是普通 Python/文档任务，可以不启动环境；
- 如果后续接入 TotalSegmentator 或其它医学模型，默认使用 `fracmed`。

## 5. 注意事项

安装时 pip 提示若干命令行脚本不在 PATH 中，例如：

```text
TotalSegmentator.exe
nnUNetv2_predict.exe
```

这不影响项目使用。需要调用时应使用绝对路径，或者通过：

```powershell
D:/python/anaconda/envs/fracmed/python.exe -m <module>
```

后续如果需要直接在终端中使用这些命令，可以先激活环境：

```powershell
D:/python/anaconda/Scripts/conda.exe activate fracmed
```
