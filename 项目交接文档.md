## 6. 启发式候选点状态

当前自动候选点主要来自规则算法，不是训练好的骨折检测模型。

典型候选来源：

- `edge`：骨窗下局部边缘突变；
- `frag`：小高密度连通区域；
- `manual`：人工添加。

候选分数只是排序分数，不是骨折概率。

相关文件：

```text
src/candidate_detection.py
src/external_models/heuristic_adapter.py
启发式算法说明.md
```

## 11. RSNA C-Spine 8th Runtime 状态

notebook 推理流程已抽取到：

```text
src/external_models/cspine_8th_runtime.py
```

已实现阶段：

- `SagSegModel`：sagittal 视角 C1-C7 定位；
- `BoneSegModel`：axial 骨结构分割；
- `SliceClsModel`：2.5D slice-level 特征提取；
- `SeqModel`：C1-C7 序列分类；
- DICOM 递归读取、z 排序；
- OpenCV/PyTorch 版 resize + pad；
- Gradio 表格输出。

为了适配 RTX 4060 Laptop GPU，第一版默认每阶段只加载 1 fold：

```text
max_folds_per_stage: 1
```

CLI probe：

```powershell
D:/python/anaconda/envs/fracmed/python.exe scripts/probe_cspine_8th_runtime.py --case 1
```

当前 Case 1 实测：

```text
device: cuda
elapsed_seconds: about 10s
fallback_used: True
fallback_reason: RuntimeError: Sagittal model did not select enough cervical slices.
```

解释：

- Case 1 不是颈椎 CT；
- sagittal C1-C7 定位阶段无法找到足够颈椎切片；
- runtime 返回 notebook 原始 fallback mean；
- Gradio 表格会标注 `fallback mean; not a case-specific prediction`；
- 这不是模型对当前手腕/足部病例的真实预测。

要验证真实 case-specific 输出，需要准备一例颈椎 CT DICOM。

## 12. Gradio 中两个 C-Spine 按钮

页面区域：

```text
External C-Spine Reference
```

`Check C-Spine Readiness`：

- 只检查代码、权重、依赖、runtime 前置条件；
- 不跑模型；
- 看到 `Runtime ready: True` 才说明可以尝试运行。

`Run C-Spine Reference`：

- 调用真实抽取后的 runtime；
- 输出 `study` 和 `C1` 到 `C7` 风险分数；
- 如果非颈椎病例触发 fallback，会明确显示不是病例特异预测。

## 15. 下一步建议

优先级较高：

1. 准备一例真实颈椎 CT DICOM，验证 RSNA C-Spine 8th 是否能输出非 fallback 的病例特异结果。
2. 评估 1 fold runtime 的显存和耗时，再决定是否开放 5 fold ensemble。
3. 如果要继续医院 demo，页面上应明确说明当前 C-Spine 是跨域模型，不用于手腕/足部诊断。
4. 若医院能提供少量真实手腕/足部 CT 标注，应优先进入数据整理和轻量训练路线，而不是继续堆跨域模型。

可选改进：

- C-Spine runtime 增加 `max_folds` 参数；
- Gradio 增加 C-Spine 耗时、fallback 原因、device 展示；
- nnInteractive mask 接入 3D 高亮；
- 增加候选缩略图和上一/下一候选按钮；
- 增加窗宽窗位预设。
